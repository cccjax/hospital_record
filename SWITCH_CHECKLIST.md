# 双设备切换清单（SWITCH_CHECKLIST）

## A 设备离开前（必须完成）
- [ ] 查看变更范围：`git status`（或等效变更列表）
- [ ] 更新 `HANDOFF.md` 的 1-7 节（目标、状态、已完成、下一步、待确认、风险、命令）
- [ ] 提交代码（未完成也建议做检查点提交）
- [ ] 推送到远程分支（默认 `main`）
- [ ] 在同一条 Codex 线程补一句接力说明：
  - `继续 hospital_record，先看 HANDOFF.md 再继续。`

## B 设备接手后（5 分钟内）
- [ ] 打开同一条 Codex 线程（不要新开）
- [ ] 拉取最新代码（`git pull`）
- [ ] 阅读 `HANDOFF.md` 的 1-6 节
- [ ] 启动并验证首页可进入，无空白（Web 或 Flutter）
- [ ] 按 `HANDOFF.md` 的“下一步（前 30 分钟）”执行

## 切换过程决策规则
- 若发现分支分叉或冲突：
  - 先暂停编码，执行 `git status` 与 `git log --oneline --decorate --graph -20`，确认主线再继续。
- 若上下文不一致：
  - 以 `HANDOFF.md` 为单一事实源；
  - 先写 3 行“当前理解”，再改代码。
- 若紧急切换且功能未完成：
  - 先做检查点提交，避免另一台设备无法复现现场。
