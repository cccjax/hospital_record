# 切换清单（SWITCH_CHECKLIST）

## A. 离开设备前（必须）
- [ ] 执行 `git status`，确认本次改动范围
- [ ] 更新 `HANDOFF.md`（至少更新“当前状态、关键文件、下一步”）
- [ ] 本地执行最小回归：
  - [ ] `flutter analyze`
  - [ ] `flutter test`
  - [ ] 关键路径手测（首页、模板、我的、数据迁移）
- [ ] 提交并推送到远端分支

## B. 接手设备后（5 分钟内）
- [ ] 打开同一仓库并 `git pull`
- [ ] 阅读 `HANDOFF.md`
- [ ] 在 `flutter_app` 执行：
  - [ ] `flutter pub get`
  - [ ] `flutter analyze`
- [ ] 启动应用快速检查首页是否正常渲染

## C. 本项目专项回归
- [ ] 首页概览区渲染正常，无布局异常
- [ ] 病人列表、入院记录、日常记录可正常增删改查
- [ ] 住院测评可新增、编辑、删除、查看详情
- [ ] 多行文本字段展示为整行（不与其他字段并排）
- [ ] 数据迁移：
  - [ ] 导出文件成功
  - [ ] 导入前能看到校验拦截（异常文件）
  - [ ] 导入前有最终确认弹窗（覆盖提示 + 摘要）

## D. 发生异常时
- [ ] 先保留现场，不要 `reset --hard`
- [ ] 保存日志与报错截图
- [ ] 在 `HANDOFF.md` 追加“问题复现步骤 + 影响范围 + 临时结论”
