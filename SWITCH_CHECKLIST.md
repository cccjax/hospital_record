# 切换清单（SWITCH_CHECKLIST）

## A. 离开设备前（必须）
- [ ] 执行 `git status`，确认本次改动范围
- [ ] 更新 `HANDOFF.md`（至少更新“当前状态、验证结果、下一步”）
- [ ] 本地执行最小回归：
  - [ ] `flutter analyze lib test`
  - [ ] `flutter test`
  - [ ] 关键路径手测（首页、模板、我的、数据迁移）
- [ ] 提交并推送到远端分支

## B. 接手设备后（5 分钟内）
- [ ] 打开同一仓库并 `git pull`
- [ ] 阅读 `HANDOFF.md`
- [ ] 在 `flutter_app` 执行：
  - [ ] `flutter pub get`
  - [ ] `flutter analyze lib test`
- [ ] 启动应用快速检查首页是否正常渲染

## C. 本项目专项回归
- [ ] 旧版 `SharedPreferences` 数据可正常迁移到 JSON 文件
- [ ] 重启 App 后病人、入院、日常记录、模板数据仍保留
- [ ] 开启/修改密码后，本地安全配置中不保存明文密码
- [ ] 旧版明文密码首次验证成功后会自动升级为哈希
- [ ] 指纹解锁仍需先开启密码保护
- [ ] 首页病人卡片底色统一，护理等级颜色仅体现在标签
- [ ] 首页概览筛选可按护理等级快速筛选病人
- [ ] 首页支持“标准/紧凑”密度切换，切换后卡片宽度与行距同步变化
- [ ] 首页卡片在标准模式下无明显底部空白，且无 `RenderFlex overflow`
- [ ] 首页卡片在平板竖屏下列数自适应，比例不失真
- [ ] 首页病人列表支持左右滑分页，分页点悬浮在页面底部（半透明圆点）
- [ ] 首页姓名与护理等级标签同一行时，姓名不会被完全挤掉
- [ ] 首页工具栏“新增/删除”按钮尺寸、圆角、阴影一致
- [ ] 首页删除按钮为红色主题
- [ ] 首页支持批量删除模式（进入、取消、多选、确认删除）
- [ ] 病人信息中的护理等级可编辑
- [ ] 字段配置中护理等级不可删除
- [ ] 字段配置中护理等级颜色可自由选色（调色盘/RGB/HEX）
- [ ] 字段配置-护理等级配置行无“颜色重置”按钮
- [ ] 字段配置-护理等级配置行中文本输入框空间充足

- [ ] 字段配置中“是否可见”图标语义正确（睁眼=当前可见，闭眼=当前隐藏）

## D. 发生异常时
- [ ] 先保留现场，不要 `reset --hard`
- [ ] 保存日志与报错截图
- [ ] 在 `HANDOFF.md` 追加“问题复现步骤 + 影响范围 + 临时结论”
